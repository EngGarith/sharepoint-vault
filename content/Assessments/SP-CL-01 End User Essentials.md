# SP-CL-01 — End User Essentials

**Purpose:** This checklist helps everyday SharePoint users reflect on whether they are using the platform safely and effectively. It covers finding content, sharing responsibly, organizing files, and recovering from mistakes. It is not a test — it is a learning tool. If you answer No to something, the remediation links at the end of each section will help you improve.

**Audience:** All staff who use SharePoint to store, share, or work with documents and information.
**Estimated time:** 10–15 minutes.
**Cadence:** Complete once, then revisit annually or after any SharePoint training.

---

## How to Use This Checklist

Mark each item honestly by replacing `[ ]` with `[x]`, or circle your answer on a printed copy.

- **Yes** — I do this consistently
- **Partial** — I do this sometimes but not always
- **No** — I do not do this, or I did not know I should
- **N/A** — This does not apply to my role or the sites I use

Use the Section Score Summary to see where you stand. Share results with your SharePoint administrator or line manager if asked.

**Criticality key:** `[CRITICAL]` = 3 pts · `[STANDARD]` = 2 pts · `[BEST PRACTICE]` = 1 pt

## Section 1 — Finding and Accessing Content

- [ ] `[STANDARD]` I know how to use **SharePoint Search** to find documents across sites I have access to. *(2 pts)*
- [ ] `[STANDARD]` When I cannot access something I need, I know who to contact — the **site owner** or my manager — rather than requesting access directly via a random link. *(2 pts)*
- [ ] `[BEST PRACTICE]` I use **filters and views** within document libraries rather than scrolling through long lists of files. *(1 pt)*
- [ ] `[BEST PRACTICE]` I know how to check **who owns a site** if I need to ask for access or report a problem. *(1 pt)*

**Section 1 Max: 6 pts**

> **Remediation:** [[Search Overview]] · [[Site Permissions]] · [[Document Libraries Overview]]

## Section 2 — Sharing and Permissions

- [ ] `[CRITICAL]` I never share content using an **"Anyone" (anonymous) link** without first confirming with my manager or site owner that this is permitted. *(3 pts)*
- [ ] `[CRITICAL]` When I share a file or folder, I use **"Specific people"** links rather than broad organization-wide links unless the content is genuinely intended for everyone. *(3 pts)*
- [ ] `[STANDARD]` I know the difference between **sharing a link** (giving access to that item) and **sharing a file by email** (attaching a copy). *(2 pts)*
- [ ] `[STANDARD]` I remove sharing access to files or folders when a project ends or a collaborator no longer needs access. *(2 pts)*
- [ ] `[BEST PRACTICE]` I check sharing settings before sending a link to confirm the recipient will have the correct level of access (view only vs. edit). *(1 pt)*
- [ ] `[BEST PRACTICE]` I know how to see **who a file or folder is shared with** and can remove access when needed. *(1 pt)*

**Section 2 Max: 12 pts**

> **Remediation:** [[External Sharing]] · [[Permissions Overview]] · [[Permission Levels]]

---

## Section 3 — File Naming and Organization

- [ ] `[STANDARD]` I save files in the **correct document library for the content type**, not in personal OneDrive or a temporary location that others cannot find. *(2 pts)*
- [ ] `[STANDARD]` I use **descriptive file names** that make it clear what the document contains without needing to open it (e.g. `Contract-Supplier-ABC-2026.docx` rather than `Document1.docx`). *(2 pts)*
- [ ] `[BEST PRACTICE]` I use **metadata columns** (e.g. project name, status, document type) to tag files rather than creating deeply nested folder structures. *(1 pt)*
- [ ] `[BEST PRACTICE]` I avoid using **special characters** (`# % & * : < > ? / \ { | }`) in file names, as these can cause sync and link issues. *(1 pt)*

**Section 3 Max: 6 pts**

> **Remediation:** [[Document Libraries Overview]] · [[Library Configuration]] · [[Columns and Metadata]]

---

## Section 4 — Version History and Recovery

- [ ] `[CRITICAL]` I know that SharePoint keeps **version history** for documents, and that I can restore a previous version if I make a mistake. *(3 pts)*
- [ ] `[STANDARD]` I use **"Edit in browser"** or **check out** a file before making major edits, rather than downloading a copy and re-uploading it (which breaks version history). *(2 pts)*
- [ ] `[BEST PRACTICE]` I know how to access the **version history panel** for a document to view or restore previous versions. *(1 pt)*
- [ ] `[BEST PRACTICE]` I know that deleted files go to the **site Recycle Bin** and can be recovered by the site owner within 93 days. *(1 pt)*

**Section 4 Max: 7 pts**

> **Remediation:** [[Versioning]] · [[Document Libraries Overview]]
